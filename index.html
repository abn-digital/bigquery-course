Módulo 1

Introducción a SQL

El lenguaje universal de los datos. Desde cero hasta escribir tus propias consultas.

¿Por qué SQL? SQL existe desde los años 70 y sigue siendo el estándar para trabajar con datos estructurados. Toda herramienta de datos —BigQuery, Snowflake, Postgres, Excel Power Query— entiende SQL. Aprender SQL es aprender a hablar el idioma de los datos.

A

El lenguaje básico

  • ✓Qué es SQL y por qué es el lenguaje de los datos
  • ✓Estructura de una query: SELECT, FROM, WHERE
  • ✓Tipos de datos: strings, números, fechas, booleanos
  • ✓Cómo leer e interpretar el resultado de una consulta

-- Mi primera query sobre clientes reales

SELECT name, country, signup_date

FROM `workshop-495617.workshop_dataset.test_customers`

WHERE signup_date >= '2024-01-01'

Regla de oro: Una query SQL se lee de adentro hacia afuera. Primero FROM (de dónde vengo), después WHERE (qué filtro), y por último SELECT (qué muestro). El orden en que escribís no es el orden en que se ejecuta.

B

Filtrar y agrupar datos

  • ✓Filtrar filas con WHERE: operadores =, AND, IN, BETWEEN
  • ✓Agrupar con GROUP BY y funciones de agregación: SUM, COUNT, AVG
  • ✓Ordenar resultados con ORDER BY
  • ✓HAVING vs WHERE: cuándo usar cada uno

-- ¿Cuántas órdenes hay por estado?

SELECT status, COUNT(*) AS total,

SUM(quantity) AS items_totales

FROM `workshop-495617.workshop_dataset.test_orders`

GROUP BY status

ORDER BY total DESC

C

Dimensiones vs. Métricas

Dimensiones

Categorías que describen los datos. No se suman.

  • → País, ciudad, canal
  • → Categoría de producto
  • → Fecha, mes, semana
Métricas

Valores numéricos que se miden y agregan.

  • → Ventas, ingresos, costo
  • → Cantidad de unidades
  • → Clics, conversiones, CTR

-- Reporte: revenue por categoría y mes

SELECT p.category, DATE_TRUNC(o.order_date, MONTH) AS mes,

SUM(o.quantity * p.price) AS revenue,

COUNT(DISTINCT o.customer_id) AS clientes_unicos

FROM `workshop_dataset.test_orders` o

JOIN `workshop_dataset.test_products` p ON o.product_id = p.id

WHERE o.status = 'completed'

GROUP BY 1, 2

ORDER BY mes DESC

D

Joins: combinar tablas

  • ✓Qué es un JOIN: combinar dos tablas por una clave en común
  • ✓INNER JOIN: solo las filas que coinciden en ambas tablas
  • ✓LEFT JOIN: todas las filas de la izquierda, aunque no haya match
  • ✓Ejemplo: cruzar tabla de clientes con tabla de transacciones

-- Clientes con sus órdenes

SELECT c.name, COUNT(o.id) AS total_ordenes,

SUM(o.quantity * p.price) AS total_gastado

FROM `workshop_dataset.test_customers` c

LEFT JOIN `workshop_dataset.test_orders` o

ON c.id = o.customer_id

LEFT JOIN `workshop_dataset.test_products` p

ON o.product_id = p.id

GROUP BY c.name

Módulo 2

El entorno de BigQuery

Conocé la plataforma de datos de Google Cloud: datasets, tablas, rutinas y el editor SQL.

¿Qué es BigQuery? Es el data warehouse de Google Cloud. Está diseñado para analizar terabytes de datos en segundos, sin servidores que administrar. A diferencia de una base de datos tradicional, BigQuery no es transaccional — está optimizado para leer y analizar grandes volúmenes, no para escribir fila por fila.

A

Proyectos, Datasets y Tablas

Proyecto

El nivel más alto. Agrupa todos los recursos y se asocia a una cuenta de facturación de GCP.

Dataset

Contenedor de tablas. Funciona como una carpeta o base de datos. Tiene región y permisos propios.

Tabla / Vista / Rutina

Donde viven los datos. Las vistas son queries guardadas. Las rutinas son funciones o procedimientos reutilizables.

-- Así se referencia una tabla en BigQuery

`proyecto`.`dataset`.`tabla`

Ejemplo real:

`mi-empresa-prod`.`ventas`.`transacciones_2024`

💡

Tip de nomenclatura: Usá guiones medios en el nombre del proyecto (impuesto por GCP) y guiones bajos en datasets y tablas. Evitá mayúsculas y caracteres especiales.

A

Crear una Table Function

Una Table Function es una rutina que recibe parámetros y devuelve una tabla. Permite encapsular lógica de consulta reutilizable — la llamás como si fuera una tabla en cualquier FROM.

  • ✓Definir parámetros de entrada: fecha inicio, estado
  • ✓La función devuelve un resultado tipo tabla que podés consultar
  • ✓Reutilizable: usala en dashboards, otras queries, o scheduled queries
  • ✓Más flexible que una vista porque acepta parámetros dinámicos

-- Table Function: órdenes por estado y período

CREATE OR REPLACE TABLE FUNCTION

`workshop-495617.workshop_dataset.fn_orders`(

start_date DATE,

order_status STRING)

AS (

SELECT o.id, c.name, p.category,

o.quantity * p.price AS revenue,

o.order_date, o.status

FROM `workshop_dataset.test_orders` o

JOIN `workshop_dataset.test_customers` c ON o.customer_id = c.id

JOIN `workshop_dataset.test_products` p ON o.product_id = p.id

WHERE o.order_date >= start_date

AND o.status = order_status

);

¿Cómo se usa la Table Function?

Llamarla directamente

SELECT * FROM

fn_orders('2024-01-01', 'completed')

Usarla en un reporte

SELECT category, SUM(revenue)

FROM fn_orders(CURRENT_DATE() - 30, 'completed')

GROUP BY category

B

Scheduled Queries

Las scheduled queries ejecutan una query automáticamente en un horario definido, sin intervención manual.

  • ✓Configurar frecuencia: cada hora, diario, semanal
  • ✓Definir destino: tabla donde se guardan los resultados
  • ✓Modo WRITE_TRUNCATE vs WRITE_APPEND
  • ✓Ver historial de ejecuciones y manejar errores con notificaciones

C

Crear una tabla externa

Una tabla externa vive en Google Sheets o Cloud Storage — BigQuery la consulta sin copiar los datos.

  • ✓Crear tabla externa apuntando a un Google Sheet
  • ✓Definir schema: nombres y tipos de columnas
  • ✓Consultar la tabla con SQL normal
  • ✓Ventaja clave: los datos se actualizan solos en el Sheet

-- Crear tabla externa desde Google Sheets

CREATE OR REPLACE EXTERNAL TABLE

`workshop-495617.workshop_dataset.ext_budgets`

(category STRING, budget FLOAT64,

period STRING)

OPTIONS (

format = 'GOOGLE_SHEETS',

uris = ['https://docs.google.com/...'],

skip_leading_rows = 1

);

✅ Tabla externa — cuándo usarla

  • → El equipo actualiza el Sheet constantemente
  • → No querés duplicar los datos
  • → Datos de referencia que cambian poco (budgets, regiones)

⚠️ Tabla nativa — cuándo conviene más

  • → Volumen grande de datos (miles de filas+)
  • → Necesitás velocidad de consulta óptima
  • → Datos históricos que ya no cambian

Generar Queries con IA

  • ✓Describís en lenguaje natural qué datos necesitás
  • ✓La IA genera el SQL completo con explicación
  • ✓Iterás con prompts: "agregale un filtro por mes"
👤 Vos:

"Dame las ventas totales por región del último mes, solo las que superan $1000"

🤖 IA genera:

SELECT region, SUM(monto) AS total

FROM `ventas.transacciones`

WHERE DATE_TRUNC(fecha, MONTH)

= DATE_TRUNC(CURRENT_DATE(), MONTH)

GROUP BY region

HAVING total > 1000

Conceptos clave para llevarte

SQL

Lenguaje para consultar bases de datos. Declarativo: describís qué querés, no cómo obtenerlo.

Dataset

Contenedor de tablas en BigQuery. Equivale a una base de datos. Tiene región y permisos propios.

Rutina

Bloque de lógica SQL guardado y reutilizable. Se invoca con CALL nombre().

Schedule

Query que se ejecuta automáticamente en un horario. Sin intervención humana.

JOIN

Combinar dos tablas usando una columna en común (clave). Permite cruzar información de fuentes distintas.

MCP

Model Context Protocol: protocolo que permite a una IA conectarse a herramientas externas como BigQuery.
